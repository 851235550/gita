# Gita MVP 测试用例说明

> 配合《Gita 任务拆解》使用。为每个关键模块提供具体的输入/预期输出，减少 AI Agent 编写测试时的自由发挥空间。测试用例编号与任务编号对应，如 `TC-T1.1-01` 对应任务 T1.1 的第 1 个用例。

---

## 1. git 模块

### TC-T1.1：Git 命令封装

| 用例       | 前置条件                                     | 输入/操作                   | 预期结果                                                        |
| ---------- | -------------------------------------------- | --------------------------- | --------------------------------------------------------------- |
| TC-T1.1-01 | 当前目录是有效 Git 仓库，无 staged 变更      | 调用 `HasStagedChanges()`   | 返回 `false`                                                    |
| TC-T1.1-02 | `git add` 一个新文件后                       | 调用 `HasStagedChanges()`   | 返回 `true`                                                     |
| TC-T1.1-03 | 当前目录不是 Git 仓库（如 `/tmp/empty_dir`） | 调用 `IsGitRepo()`          | 返回 `false`，不 panic                                          |
| TC-T1.1-04 | 当前目录不是 Git 仓库                        | 调用 `GetStagedDiff()`      | 返回 error，错误信息包含"不是有效的 Git 仓库"关键字             |
| TC-T1.1-05 | staged 一个修改了 3 行的文件                 | 调用 `GetStagedDiff()`      | 返回内容包含标准 unified diff 格式（`diff --git`、`@@` 等标记） |
| TC-T1.1-06 | staged 一个二进制文件（如 `.png`）           | 调用 `GetStagedDiff()`      | 返回内容中该文件显示为 `Binary files differ`，不报错、不崩溃    |
| TC-T1.1-07 | staged 3 个文件                              | 调用 `GetStagedFileNames()` | 返回长度为 3 的字符串数组，路径与实际一致                       |

### TC-T1.2：Hunk Header 解析

| 用例       | 输入（diff 片段）                                                                                             | 预期输出                                                     |
| ---------- | ------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
| TC-T1.2-01 | Go 文件 diff，hunk header 为 `@@ -10,7 +10,8 @@ func doSomething(ctx context.Context) error {`                | 提取到上下文 `func doSomething(ctx context.Context) error {` |
| TC-T1.2-02 | Python 文件 diff，hunk header 为 `@@ -5,3 +5,4 @@ def calculate_total(items):`                                | 提取到上下文 `def calculate_total(items):`                   |
| TC-T1.2-03 | JS 文件 diff，hunk header 为 `@@ -20,5 +20,6 @@ export function handleClick() {`                              | 提取到上下文 `export function handleClick() {`               |
| TC-T1.2-04 | 冷门语言（如 `.proto` 文件，Git 无内置 funcname 规则），hunk header 为 `@@ -1,2 +1,3 @@` （末尾无上下文文本） | 该 hunk 返回空字符串，不报错                                 |
| TC-T1.2-05 | 同一文件内 3 个 hunk 落在同一函数内（上下文文本相同）                                                         | 结果去重后该文件仅保留 1 条上下文记录                        |
| TC-T1.2-06 | 传入完全不含 `@@` 标记的字符串（异常输入）                                                                    | 返回空 map，不报错                                           |

## 2. context 模块

### TC-T2.1：Diff 分级边界值

**这是最容易出 Bug 的地方，边界值必须精确覆盖：**

| 用例       | 输入行数                                                    | 预期分级                                                                   |
| ---------- | ----------------------------------------------------------- | -------------------------------------------------------------------------- |
| TC-T2.1-01 | 0                                                           | Small                                                                      |
| TC-T2.1-02 | 999                                                         | Small                                                                      |
| TC-T2.1-03 | 1000                                                        | Medium（含边界）                                                           |
| TC-T2.1-04 | 4999                                                        | Medium                                                                     |
| TC-T2.1-05 | 5000                                                        | Large（含边界，注意与 config `max_diff_lines` 默认值一致）                 |
| TC-T2.1-06 | 14999                                                       | Large                                                                      |
| TC-T2.1-07 | 15000                                                       | ExtraLarge（含边界）                                                       |
| TC-T2.1-08 | 15001                                                       | ExtraLarge                                                                 |
| TC-T2.1-09 | config 中 `max_diff_lines` 被用户改为 2000 时，输入 1500 行 | 分级逻辑需按用户配置动态调整（此时应为 Medium 而非 Small），不能硬编码阈值 |

> **实现约定**：所有边界值判断使用"左闭右开"还是"左开右闭"需在代码注释中明确写出，避免后续维护者猜测。建议统一为 `< 1000` / `1000 ~ 4999` / `5000 ~ 14999` / `>= 15000`。

### TC-T2.2：摘要压缩（抓大放小）

| 用例       | 输入                                                                    | 预期输出                                                                                               |
| ---------- | ----------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| TC-T2.2-01 | 一次 commit 中，文件 A 改动 30 行，文件 B 改动 800 行，整体判定为 Large | 输出中文件 A 为完整 diff 内容；文件 B 为"路径+变更类型+行数统计+hunk 上下文列表"的摘要形式             |
| TC-T2.2-02 | 文件改动恰好 50 行                                                      | 按需求文档"< 50 行"的边界定义，50 行本身不满足"< 50"，应走摘要压缩路径（需在代码中明确边界，避免歧义） |
| TC-T2.2-03 | 文件为新增文件（无历史内容，即整个文件都是新增）                        | 变更类型标注为"新增"，摘要中包含新文件的 hunk 上下文（如有）                                           |
| TC-T2.2-04 | 文件被删除                                                              | 变更类型标注为"删除"，不产出 hunk 上下文（因无新增内容可解析）                                         |
| TC-T2.2-05 | 文件被重命名且内容无变化                                                | 变更类型标注为"重命名"，摘要中体现旧路径 → 新路径                                                      |

### TC-T2.3：Context Builder 整体输出

| 用例       | 分级                       | 预期发送内容组成                                            |
| ---------- | -------------------------- | ----------------------------------------------------------- |
| TC-T2.3-01 | Small                      | 仅完整 diff，不包含 stat 或 name-only                       |
| TC-T2.3-02 | Medium                     | 完整 diff + `--stat` 输出                                   |
| TC-T2.3-03 | Large                      | 摘要压缩结果（不含原始完整 diff，除非单文件命中"抓大放小"） |
| TC-T2.3-04 | ExtraLarge，未传 `--force` | 阻塞等待用户确认，输出需求文档 6.4 节提示文案               |
| TC-T2.3-05 | ExtraLarge，传入 `--force` | 跳过确认，直接使用 `--stat` + `--name-only` 生成            |

## 3. prompt 模块

### TC-T1.3：模板加载与渲染

| 用例       | 前置条件                                   | 预期结果                                                                                             |
| ---------- | ------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| TC-T1.3-01 | `~/.gita/prompts/commit.md` 不存在         | 使用内置默认模板，不报错                                                                             |
| TC-T1.3-02 | `~/.gita/prompts/commit.md` 存在且内容合法 | 优先使用用户自定义模板                                                                               |
| TC-T1.3-03 | 模板中包含 `{{diff}}{{hint}}{{language}}`  | 三个变量均被正确替换为传入值                                                                         |

| 用例       | 操作                                   | 预期结果                                                                         |
| ---------- | -------------------------------------- | -------------------------------------------------------------------------------- |
| TC-T4.1-01 | LLM 返回结果后，用户直接回车（默认 Y） | 执行 `git commit -m "<生成内容>"`，退出码 0                                      |
| TC-T4.1-02 | 用户输入 `n`                           | 不执行任何 git 命令，进程正常退出，退出码 0（用户主动取消不算错误）              |

## 8. 测试数据 Fixture 建议

为避免每个 Agent 各自造轮子导致 fixture 不一致，建议统一维护：

```text
test/fixtures/
├── diffs/
│   ├── small.diff          # < 1000 行，纯文本改动
│   ├── medium.diff         # 1000~5000 行
│   ├── large.diff          # 5000~15000 行，含大小文件混合
│   ├── extra_large.diff    # > 15000 行
│   ├── binary.diff         # 含二进制文件
│   └── multi_lang.diff     # Go/Python/JS 混合，用于 hunk 解析测试
├── configs/
│   ├── valid_config.yaml
│   ├── missing_provider.yaml
│   └── empty_config.yaml
└── templates/
    └── custom_commit.md
```

所有涉及 diff 内容的测试用例，优先复用此处 fixture，而非在测试代码中内联长字符串，便于后续维护与复查。
